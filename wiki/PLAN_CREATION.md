Create a plan for implementing a code change. This may include adding a new feature, modifying behavior, or resolving incorrect behavior.

The plan should contain:

1. Current behavior analysis
   - What the system does now (relevant to the task)

2. Desired behavior
   - What the system should do after the change

3. Gap analysis
   - What differences exist between current and desired behavior

4. Detailed implementation steps
   - What functions/files to modify or create
   - What exactly should be changed
   - What must remain unchanged

5. Risks
   - Potential regressions or unintended side effects

6. Success criteria
   - Clear conditions that confirm the change works as intended

7. Testing strategy
   - How to verify correctness and prevent regressions

Rules:
- Do not implement code
- Prefer minimal and targeted changes
- Do not introduce unrelated refactors or cleanup
- Preserve existing behavior outside the scope of the task
- If something is unclear or ambiguous, ask before proceeding
