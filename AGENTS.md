# Agent Information: ngx-testbox Library

## Project Overview
This is the **ngx-testbox** Angular library source — an integration testing utility for Angular applications. It provides black-box testing helpers including async stabilization (`runTasksUntilStable`), HTTP mocking instructions, and DOM harnesses.

## Workspace Layout
- **Library source:** `projects/ngx-testbox/src/lib/` (runtime code: directive, consts)
- **Testing public API:** `projects/ngx-testbox/testing/src/` (the code consumers import from `ngx-testbox/testing`)
- **Library unit tests:** `projects/ngx-testbox/src/__tests__/` (tests for the testing utilities themselves)
- **Example application:** `projects/tour-of-heroes/` (demonstrates library usage in real component specs)
- **Built output:** `dist/ngx-testbox/` (generated by ng-packagr)

## Technology Stack
- Angular **v20.3.15**
- TypeScript **~5.9.3**
- Build tool: `@angular-devkit/build-angular:ng-packagr`
- Test runner: **Karma + Jasmine**
- Test browser: Chrome (headless in CI)
- Currently depends on **zone.js** for testing

## Key Commands
Run these from the **repository root** (`/home/user/ngx-testbox`):

```bash
# Build the library (development)
npm run build:dev:lib

# Build the library (production)
npm run build:prod:lib

# Run tests (watch mode in Chrome)
npm test

# Run tests once with coverage in headless Chrome
npm run test:coverage

# Serve the example application
npm start
```

## Entry Points
The library exposes two entry points configured via `paths` in `tsconfig.json`:
- `ngx-testbox` — main runtime exports (e.g., `TestIdDirective`)
- `ngx-testbox/testing` — testing utilities (e.g., `runTasksUntilStable`, `DebugElementHarness`, `completeHttpCalls`)

The physical source for `ngx-testbox/testing` is under `projects/ngx-testbox/testing/src/`.

## Testing Conventions
- Spec files use the `.spec.ts` suffix.
- Karma config is implicit via Angular CLI; no custom `karma.conf.js` edits are typically needed.
- The example app (`tour-of-heroes`) serves as integration tests. If you modify the testing API, ensure those specs still compile and pass.
- `projects/ngx-testbox/tsconfig.spec.json` includes both `src` and `testing` rootDirs for spec compilation.

## Code Style & Constraints
- `strict: true` and all Angular strict compiler flags are enabled.
- Target is `ES2022`, module is `ES2022`, moduleResolution is `bundler`.
- Prefer **standalone components** (the example app already uses them).
- Do not add heavy external dependencies to the library; keep the peer dependency surface minimal (`@angular/common`, `@angular/core` >= 9).

## When Modifying This Library
1. Update **both** the library source and the `tour-of-heroes` example tests if the public API changes.
2. Run `npm run test:coverage` before considering a change complete.
3. If you add a new public export, update `projects/ngx-testbox/testing/src/public_api.ts` (and the main `public_api.ts` if applicable).
4. The library package version lives in `projects/ngx-testbox/package.json` (not the root `package.json`).
